# [TASK007] - 透群友联动系统

**Status:** In Progress
**Added:** 2026-03-04
**Updated:** 2026-03-06
**Origin:** 从 TASK001 拆分
**Depends:** TASK004（负值世界基础）, TASK006（技能系统，四/五阶增强）

## Original Request
实现透群友/榨群友与核心数值（长度/深度）的联动，包括正值透→自己+长度、负值榨→深度比较机制（食物链），以及技能增强叠加。

## 已确认设计

### 基础约束（已由现有代码实现 ✅）

- **透群友**：仅正值世界（男）可发起 → `positive_world_guard`
- **榨群友**：仅负值世界（女）可发起 → `negative_world_guard`
- **榨群友目标**：必须是正值世界（男）玩家 → `positive_target_guard`
- **目标选择**：群友模式支持 @指定 → `_select_member` 尊重 @ 参数

### 联动效果设计讨论过程

#### 方案 A：联动长度/深度（原 TASK001 食物链设计）

| 行为 | 对自己 | 对目标 |
|------|--------|--------|
| 透群友 | +0.1~0.3cm | 注入效果不变 |
| 榨群友 | 侵蚀/汲取 | 侵蚀 -0.1~0.5cm / 汲取无 |

**评价**：设计完整但复杂（需实现深度比较纯函数）

---

#### 方案 B：联动 PK 胜率

| 行为 | 对自己 | 对目标 |
|------|--------|--------|
| 透群友 | +1% 胜率 | 无 |
| 榨群友 | +1% 胜率 | -1% 胜率 |

**讨论**：胜率只对同性 PK 有效（击剑=正值间、磨豆腐=负值间），透/榨都是跨性别 → 自己加胜率对即将对抗的对手无用

**被否决理由**：
1. 胜率收益在跨性别对抗中无意义
2. 与未来女性设定冲突：女性可直接吞噬男方长度转化为男生 → 联动必须是长度维度

---

#### 方案 C：转移长度机制 ✅ **确认**

| 行为 | 对自己 | 对目标 | 主题 |
|------|--------|--------|------|
| **透群友** | +0.1~0.3cm 长度 | 保持注入效果不变 | 征服 = 自身成长 |
| **榨群友** | +0.1~0.5cm 长度（负数变小，向零靠拢） | -0.1~0.5cm 长度 | 吞噬 = 偷取对方长度为己用 |

**关键机制**：
- 榨群友 = 转移长度（女方偷取男方长度）
- 负值女性长期榨取可以逐步回归正值世界（从 -10cm → -5cm → -1cm → +2cm）
- 为未来"吞噬转性"机制奠定基础
- 透群友中如果发起者是 xnn（被反透情况），不获得自身长度加成

**平衡考量**：
- 透获益少（0.1~0.3）：男方获取长度渠道已充分（打胶、嗦、PK）
- 榨获益多（0.1~0.5）且削弱目标：补偿进入负值世界的代价
- 榨只能选正值目标：目标池受限 ✅
- 两个指令都有 CD 限制（3600s）：一天最多 24 次 ✅

### 与技能的叠加（需 TASK006 完成后）

- 正值四阶"征服光环"：透群友效果提升
- 负值四阶"侵蚀之力"：榨群友效果提升
- 正值五阶"群体碾压"：可以透多人
- 负值五阶"深渊召唤"：可以榨多人

### 设计决策
- D12: 榨群友效果 = 转移长度（女方吸取男方长度）✅
- D18: 透群友需要与长度联动 ✅
- D19: 榨群友联动机制（单向转移 vs 深度比较） → 确认为单向转移 ✅
- D20: 透群友联动方向 → 确认为自身长度增长（+0.1~0.3cm） ✅
