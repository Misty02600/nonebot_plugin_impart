## Original Request
在所有功能系统实现完毕后，进行系统性数值平衡测试、经济健康度验证、数据库迁移和 UI 适配。

## 范围

### 一、数值平衡测试
- 各阶级触发/目标数值最终确定（TASK005 中的占位数值）
- 技能数值效果微调（TASK006）
- Buff 叠加极端场景测试（TASK008）
- 经济循环健康度：打胶/PK/透/嗦 各途径的通胀/通缩速率
- 模拟不同活跃度玩家的生命周期（日活跃 vs 周活跃 vs 低活跃）

### 二、模拟测试环境
- 扩展 `tools/pk_simulation.py` 或新建模拟工具
- 模拟 N 个玩家在 M 天内的游戏进程
- 输出统计：平均到达各阶级时间、长度分布、通胀率
- 验证关键指标：
  - 一阶：几天内可达
  - 五阶：≥1年
  - 负值世界与正值世界达成速率对称

### 三、数据库迁移
- 现有数据库 schema 到新 schema 的迁移脚本
- 现有用户数据兼容（长度/胜率字段保留）
- 新增字段：阶级状态、技能、成就、连击计数、运势等
- 迁移测试（保证无数据丢失）

### 四、UI 适配
- 查询命令显示：称号、技能、深度/长度、成就数
- 排行榜适配：正负统一排列、称号显示
- 帮助信息全面更新（新增所有指令说明）
- 挑战/雌堕/称号获得的播报文案最终审定

### 五、集成测试
- 端到端测试：完整的游戏流程覆盖
- 边界情况：雌堕边界、阶级边界、吸取回正、Buff 叠加极端值
- 性能测试：大量用户时的 DB 查询效率
