**Status:** Pending

## Original Request
实现永久成就解锁系统，达成里程碑后获得不可失去的奖励（被动加成或装饰称号），作为长期激励机制。

## 已确认设计（继承自 TASK001）

### 设计原则
- 成就一旦完成就**永不失去**（与阶级称号不同）
- 奖励分三类：被动加成、装饰称号、一次性奖励
- 作为长期目标，激励持续参与

### 成就列表

| 成就名     | 条件                       | 奖励                         | 类型   |
| ---------- | -------------------------- | ---------------------------- | ------ |
| 🏆 百战不殆 | 累计 PK 胜利 100 次        | PK 基础获取永久 +5%          | 被动   |
| 🤕 屡败屡战 | 累计 PK 失败 100 次        | PK 基础损失永久 -5%          | 被动   |
| 💉 注入大师 | 单日被注入量超过 500ml     | 永久获得注入量 +10% 的加成   | 被动   |
| 🔄 涅槃重生 | 使用过"吸取回正"至少 1 次  | 永久获得装饰称号前缀"重生の" | 装饰   |
| 👥 社交之王 | 累计透/榨过 50 个不同用户  | 透/榨群友长度效果 +0.1cm     | 被动   |
| 🏔️ 登峰造极 | 完成正值三阶挑战           | 查询时显示特殊动画效果       | 装饰   |
| 🕳️ 深渊归来 | 从负值转回正值（任意方式） | 初始长度从 10 提升到 15      | 一次性 |
| 🔥 不屈之人 | 挑战失败 3 次后再次成功    | 该阶级被动 buff 额外 +5%     | 被动   |

### 与其他系统的关联
- "涅槃重生"需要 TASK006 的吸取回正
- "登峰造极"需要 TASK005 的多阶级系统
- "社交之王"需要 TASK007 的透/榨联动
- "深渊归来"需要 TASK004 的负值世界
- 其余成就仅依赖现有 PK/注入系统

## Implementation Plan

- [ ] 9.1 成就数据模型
  - 成就定义表（成就ID、名称、条件类型、条件参数、奖励类型、奖励参数）
  - 用户成就记录表（用户ID、成就ID、完成时间）
  - 成就进度跟踪（如 PK 胜利次数、不同用户数等累计值）
- [ ] 9.2 成就检测框架
  - 在各操作后触发成就检测
  - 条件匹配引擎（累计值、比较值、存在性检查）
  - 避免每次操作都遍历全部成就（优化：按触发事件类型分组）
- [ ] 9.3 成就奖励应用
  - 被动加成类：注入到 Buff 系统（TASK008）中作为永久 buff
  - 装饰类：在查询/排行榜显示中添加装饰元素
  - 一次性类：达成时立即应用效果
- [ ] 9.4 成就通知与展示
  - 达成成就时的播报（弹幕风格提示）
  - `查看成就` 指令：显示已完成和未完成成就列表
  - 成就进度显示（如 "PK胜利 67/100"）
- [ ] 9.5 成就可扩展性
  - 确保框架支持后续新增成就
  - 考虑成就类别分组（战斗、社交、探索等）
