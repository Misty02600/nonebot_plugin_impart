# [TASK002] - PK 胜率机制重设计

**Status:** Completed
**Added:** 2026-03-03
**Updated:** 2026-03-05

## Original Request

当前 PK 胜率机制存在两个核心问题：
1. **P1**: 胜负判定只看发起方 `win_probability`，完全忽略对方
2. **P2**: 固定 ±0.01 无上下限保护，理论可无限漂移

目标：最小改动修复这两个问题，同时增加赌博心理（爆冷奖励），保留原系统的简洁性。

## Design (最终版 v2)

### 设计哲学

- **不过度设计**：不引入 sigmoid、长度对比、零期望公式等复杂机制
- **保留胜率属性**：`win_probability` 是群友可感知的成长维度
- **赌博心理**：弱方赢了拿更多（彩票型），强方稳赢少赚（稳健型）
- **强方有回报**：高胜率的人 PK 期望为正，胜率高有用
- **刺激的胜率变化**：中心区域变化大，边界自然减速，不会卡死

### 1. 胜率判定：归一化

$$P(A\text{赢}) = \frac{W_A}{W_A + W_B}$$

- 双方胜率都参与，修复 P1
- 群友一看就懂："我 0.6 对方 0.4，我赢面 6:4"
- 等胜率时和原系统完全一致（50%）
- 常见胜率范围 (0.4–0.6) 与原版差异极小

### 2. 奖惩公式：rake + 爆冷倍率

$$\text{bonus}_X = \max\!\left(1,\; \frac{W_{\text{opponent}}}{W_X}\right)$$

$$\text{赢家获得} = \text{num} \times r \times \text{bonus}_{\text{winner}}$$
$$\text{输家损失} = \text{num}$$

- `r`：可配置 rake 率（默认 **0.5**，与原系统一致：`num/2`）
- `num`：`get_random_num()` 结果（期望 0.6）
- **bonus 天然只有弱方 > 1**（强方 bonus 永远等于 1）
- **只加赢不加输**：输家固定损失 num，不受倍率影响 → 彩票型赌博

### 3. 胜率调整：单边抛物线阻尼

$$\Delta W_{\text{raw}} = \begin{cases} -K & \text{赢了（降胜率）} \\ +K & \text{输了（升胜率）} \end{cases}$$

$$f(W) = 4W(1-W) \qquad \text{(抛物线阻尼函数)}$$

**核心规则——单边阻尼**：
- 往边界方向走（赢了且 W<0.5，输了且 W>0.5）→ 应用阻尼：$\Delta W = \Delta W_{\text{raw}} \times f(W)$
- 往中心方向走（赢了且 W>0.5，输了且 W<0.5）→ **全速不阻尼**：$\Delta W = \Delta W_{\text{raw}}$

**参数**：
- $K = 0.02$（可配置，中心处有效变化率为原系统 2 倍）
- 无需 clamp——抛物线 $f(W) \to 0$ 当 $W \to 0$ 或 $W \to 1$，数学上不可能到达边界

```python
def calc_win_rate_delta(w: float, won: bool, k: float = 0.02) -> float:
    raw = -k if won else k  # 赢了降，输了升
    # 判断方向：raw<0且w<0.5 = 往0走；raw>0且w>0.5 = 往1走
    toward_boundary = (raw < 0 and w < 0.5) or (raw > 0 and w > 0.5)
    if toward_boundary:
        return raw * 4.0 * w * (1.0 - w)  # 阻尼
    return raw  # 全速
```

**设计意图**：
- 用户要求"变化幅度大更刺激，超出范围后压缩"
- 用户洞察：胜率低时下降应该减速，但上升不应该减速（反之亦然）
- 与 P 和对手无关，仅取决于自身当前胜率和本局输赢

### 4. 挑战系统：保持原结构不变

原系统 entry=25, target=30, fail_line=25, debuff=×0.8 的结构保留。

## 具体例子（r = 0.5, num 取期望 0.6）

### 例1：势均力敌 (Wa=0.50, Wb=0.50)

| | P(赢) | bonus | 赢了拿 | 输了亏 | 每 PK 期望 |
|---|------|-------|--------|--------|-----------|
| A | 50% | ×1.0 | 0.30 | 0.60 | **-0.15** |
| B | 50% | ×1.0 | 0.30 | 0.60 | **-0.15** |

双方都是负 EV（与原系统一致）。PK 纯粹是赌博刺激。

### 例2：明显优势 (Wa=0.60, Wb=0.40)

| | P(赢) | bonus | 赢了拿 | 输了亏 | 每 PK 期望 |
|---|------|-------|--------|--------|-----------|
| A(强) | 60% | ×1.0 | 0.30 | 0.60 | **-0.06** |
| B(弱) | 40% | ×1.5 | 0.45 | 0.60 | **-0.18** |

强方仍是负 EV 但亏损小，弱方万一赢了拿 0.45 > 正常 0.30 → 赌博心理。

### 例3：碾压 (Wa=0.70, Wb=0.35)

| | P(赢) | bonus | 赢了拿 | 输了亏 | 每 PK 期望 |
|---|------|-------|--------|--------|-----------|
| A(强) | 66.7% | ×1.0 | 0.30 | 0.60 | **+0.001** |
| B(弱) | 33.3% | ×2.0 | 0.60 | 0.60 | **-0.200** |

正好在正期望门槛。弱方赢一把拿 0.60 = 正常的 2 倍！"万一呢？"

### 例4：弱方主动打强方 (Wa=0.35, Wb=0.65)

| | P(赢) | bonus | 赢了拿 | 输了亏 | 每 PK 期望 |
|---|------|-------|--------|--------|-----------|
| A(弱) | 35% | ×1.86 | 0.56 | 0.60 | **-0.194** |
| B(强) | 65% | ×1.0 | 0.30 | 0.60 | **-0.015** |

弱方明知亏也敢打，因为赢了拿 0.56 远超正常。赌的是爆冷。

### 强方正期望条件

$$P_A > \frac{1}{r+1}$$

r=0.5 时门槛为 66.7%（Wa:Wb = 2:1）。高门槛意味着只有真正的强方才有正期望。

### 胜率变化速度表 (K=0.02)

| 当前 W | f(W)=4W(1-W) | 往边界走 ΔW | 往中心走 ΔW |
|--------|-------------|-------------|-------------|
| 0.50   | 1.000       | ±0.0200     | ±0.0200     |
| 0.45   | 0.990       | ±0.0198     | ±0.0200     |
| 0.40   | 0.960       | ±0.0192     | ±0.0200     |
| 0.30   | 0.840       | ±0.0168     | ±0.0200     |
| 0.20   | 0.640       | ±0.0128     | ±0.0200     |
| 0.10   | 0.360       | ±0.0072     | ±0.0200     |
| 0.05   | 0.190       | ±0.0038     | ±0.0200     |

中心附近变化大（比原 ±0.01 快 2 倍），边界区域自然减速，回中心方向始终全速→符合"输多了容易回血"的直觉。

## 数学推导

### 弱方期望（Wa < Wb，bonus = Wb/Wa）

$$E[\Delta L_A] = \frac{W_A}{W_A+W_B} \cdot \text{num} \cdot r \cdot \frac{W_B}{W_A} - \frac{W_B}{W_A+W_B} \cdot \text{num} = \frac{\text{num} \cdot W_B}{W_A+W_B}(r-1)$$

r < 1 时**始终为负**。弱方没有套利空间。

### 强方期望（Wa ≥ Wb，bonus = 1）

$$E[\Delta L_A] = \frac{\text{num}}{W_A+W_B}(W_A \cdot r - W_B)$$

正期望条件：$W_A \cdot r > W_B$，即 $\frac{W_A}{W_B} > \frac{1}{r}$。

### 阻尼收敛性

往边界方向：$\Delta W_n = K \cdot 4W_n(1-W_n)$。当 $W \to 0$ 或 $W \to 1$ 时 $\Delta W \to 0$，类似 logistic 微分方程，收敛但永远到不了边界。

往中心方向：$\Delta W = K$ 常数，线性回归。从任意极端位置回中心的速度恒定→不卡。

### 设计属性

| 属性 | 效果 |
|------|------|
| 高胜率 vs 低胜率 | 强方正期望 — 胜率高有用 ✓ |
| 等胜率 | 双方微亏 — PK 是赌博 ✓ |
| 弱方动机 | 爆冷倍率 — "万一赢了赚翻" ✓ |
| 系统经济 | r<1 → 总体负和 → 不通胀 ✓ |
| 胜率刺激 | 中心变化大（2x原版），赢了/输了感知明显 ✓ |
| 边界保护 | 往边界走自然减速，无需 clamp ✓ |
| 回血速度 | 往中心回全速，不被边界卡住 ✓ |
| 简洁性 | 3 个配置参数 (K, r, 函数形式)，公式一目了然 ✓ |

## Thought Process (设计演变记录)

设计经过多轮讨论后大幅简化。早期方案（sigmoid + 长度对比 + 零期望公式 + 多参数挑战系统）过于复杂，维护困难，群友也无法感知胜率计算。最终回归极简设计：

1. **R1-R3**: 探索废弃 win_probability（方案 C）→ 混合胜率 sigmoid(k1·ln + k2·ΔW)（方案 D）
2. **R5-R7**: 零期望公式 T=base×2(1-P)、PK=负期望赌博哲学、rake 替代硬限制
3. **R8**: 蒙特卡洛模拟验证（创建 tools/pk_simulation.py）
4. **R9**: 用户反馈方案 D 太复杂，难以维护和理解
5. **R10**: 回归简单归一化 + 爆冷倍率
6. **R11**: 胜率调整从 Elo K(E-S) 简化为 ±K（去除 P 依赖——用户"不明白为什么和P有关"）
7. **R12**: 引入抛物线阻尼 f(W)=4W(1-W)，用户要求"变化大更刺激，超出范围压缩"
8. **R13**: 用户关键洞察——单边阻尼："胜率低时下降应减速，但上升不应减速"
9. **R14**: 确定 K=0.02（中心 2x 原版），r=0.5（群友已适应原系统 num/2）

**已废弃的设计**（留作参考，不实现）：
- sigmoid(k1·ln(Sa/Sb) + k2·(Wa-Wb)) → 替换为 Wa/(Wa+Wb)
- 零期望公式 base×2(1-P)×r → 替换为 num×r×bonus
- Elo 式 K(E-S) → 替换为 ±K + 单边阻尼（去除 P 依赖）
- clamp [0.25, 0.75] → 不需要（抛物线数学上防止到达边界）
- challenge_r per-tier → 保留原挑战结构
- 对手冷却/长度差封顶 → 不需要
- k1, k2, K_ADJ, β, challenge_r[5], dist_steps, buffer_steps 等参数 → 全删
